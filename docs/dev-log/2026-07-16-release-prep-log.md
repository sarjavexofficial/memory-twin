## 今日の最重要教訓（再発防止）

### 1. EASは PC の .env を読まない
- `eas update --environment production` / EASクラウドビルドは、**EASサーバー上の環境変数**を使う。
  ローカル`.env`は無視される（`.env`はgitignoreのためビルドアーカイブにも入らない）
- 0.1.6配信とTestFlightビルド6が接続エラーだった根本原因
- **対処済み**: EASのproduction環境に4変数を登録（SARJAVEX_API_URL/APP_KEY, SUPABASE_URL/ANON_KEY）。
  以後の build/update には自動で入る。配信後のgrep検証も習慣化

### 2. Gemini停止事件（本日最大のインシデント）
- 15時頃までは正常 → 夕方、Gemini APIが `403 PERMISSION_DENIED: "Your project has been denied access"` に
- **原因**: 無料枠のAPIキーを中継サーバー（Render＝データセンターIP）経由で使う構成が、
  Googleの不正利用検知（無料キーの又貸しパターン）に誤検知された可能性が高い
- **決定**: 課金有効化したキーに切替（従量課金・基本料0円。flash-lite＋月20回上限なら月数十〜数百円）
- **本日の進捗**: ¥1,500のデポジット支払い完了（本人確認・返金可能・残高充当）。
  ただし停止直後の連続操作でアカウントに警戒フラグが付き、新規プロジェクト作成が
  「The request is suspicious」で終日ブロック。Cloud Console認証情報ページはSecure by Default
  ポリシーでAPIキー作成不可（あれはGemini用の場所ではない）
- **次の一手（2026-07-17 18:00 予定）**: AI Studio（aistudio.google.com/apikey）で
  「APIキーを作成」→ プロジェクト選択（無ければ新規作成・一晩明ければ通る見込み）→
  Render の `GEMINI_API_KEY` 差し替え → Claude が実測確認
- **予算アラート（¥1,000/月）は未設定** → キー発行が通ったら同プロジェクトに設定する

### 3. コスト3重ガード（sarjavex-api、デプロイ済み: b65e0d2）
